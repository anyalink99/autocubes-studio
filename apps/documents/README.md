# Пайплайн документов Autocubes Studio

Раздел работает по адресу `/documents.html` и является основной точкой создания документов студии.

## Поток работы

1. Пользователь выбирает шаблон в «Новый документ».
2. Документ создаётся из `templates.ts` и сохраняется локально в `autocubes-documents-v2`.
3. Поля редактируются прямо на листе; версии, комментарии и индекс документов сохраняются вместе с документом.
4. Кнопка `PDF` отправляет готовый HTML в `POST /api/documents/pdf`.
5. Локальный Chrome печатает документ с фиксированной A4-геометрией и возвращает готовый PDF на скачивание.
6. JSON остаётся переносимым источником данных, а RU/EN ZIP — пакетом для обычных двуязычных шаблонов.

## Профили сотрудников

- Модель данных, исходный профиль и печатный HTML: `employee-profile.ts`.
- Карточка шаблона и создание документа: `templates.ts`.
- Интерактивный A4-редактор: `DocumentsApp.tsx`.
- Экранные стили: `documents-enhancements.css`.
- Фотографии команды: `public/assets/team/`.
- Логотип: `public/assets/brand/autocubes.svg`; в PDF используется встроенная SVG-версия, поэтому знак не зависит от внешнего файла.

Чтобы создать следующий профиль, откройте «Документы» → «Новый документ» → «Профиль сотрудника», замените данные и нажмите `PDF`.

## Проверка

```powershell
npm run typecheck
npm run build
```

Для PDF требуется запущенный dev-сервер Studio. Браузер выбирается через `launchStudioBrowser`: сначала Playwright Chromium, затем системный Chrome.
